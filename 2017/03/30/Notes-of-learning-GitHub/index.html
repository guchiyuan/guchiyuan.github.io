Notes of learning GitHub

今天终于好好研究了一下GitHub和Git
有了GitHub之后其实，并不一定要学会Git command lines 因为GitHub提供界面化客户端 GitHub Desktop（或者SourceTree）,有了这些客户端，能很方便的进行文档版本控制了。
首先在本地建立一个workspace。进行一个新project的时候，首先建立一个repository（类似笔记本），在里面upload content，类似 html文件，你的程序等等。多人同时合作一个项目时，每个人首先fork（类似授权拷贝）原始版本，然后clone到自己本地进行修改。修改完之后，在GitHub Desktop里New Branch，相当于展现你做的贡献，提供的new features。之后你可以看到你的branch与原来master branch diff的对比。之后，需要create a pull request，然后合作者review之后觉得可以，就可以merge你的branch到master branch上面了。
如果会Git可以直接Command lines 进行操作，看上去屌屌的。基本的 init，add，push，pull， commit

tips: cd ~blog/ cd后面一定要有space
